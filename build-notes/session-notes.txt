Gotchas For The Next Claude

home.js must call buildHomeSchemas() from seo.js. Person + Org + WebSite JSON-LD were removed from index.html. Without this, the home page has zero structured data for Google.
home.js must call setBlogRows(rows) from blog.js after its parallel fetch resolves. This prevents the blog list from double-fetching when the user navigates home → /blog. The pattern: const [featuredRows, allBlogRows] = await Promise.all([...]); setBlogRows(allBlogRows);
home.js loads css/featured.css lazily — await loadCSS('/css/featured.css') before calling renderFeaturedPosts(). It should also call removeSchemas() from seo.js before buildHomeSchemas() to clean any leftover schemas from a prior route.
js/data/chord-shapes.js is a pure data file. Exports a single CHORD_SHAPES object. Keys are chord names ('G', 'Em', 'Cmaj7', 'Dsus4', 'Cadd9', 'A5' etc.). Values are arrays of finger position data used by the SVG diagram renderer in chords.js. No functions, no imports.
Chord diagram SVG spec (from master prompt Section 4): 80px × 100px, 5 frets × 6 strings, --accent for dots, --text for strings/frets. Open string = small circle above nut. Muted = X above nut. Barre = filled rounded rect. Finger number inside dot. Fret number shown if chord starts above fret 1.
css/chords.css must include a fixed bottom controls bar for mobile. On mobile, the transpose/font-size/scroll-speed controls must be position: fixed; bottom: 0; left: 0; right: 0 and the tab content container must have sufficient padding-bottom to avoid being obscured. On desktop these controls stay inline in the page flow.
Chord diagram popovers need both hover and tap. Desktop: mouseenter/mouseleave. Mobile: tap to open, tap outside or tap same chord to close. pointer-events or @media (hover: none) can help differentiate. No hover-only interactions.
Transpose logic must handle enharmonic equivalents. C# = Db, F# = Gb etc. Chord name parser must split root from suffix (Em → root E + suffix m; Cmaj7 → root C + suffix maj7; Dsus4 → root D + suffix sus4). Transpose the root up/down the chromatic scale and reattach the suffix. The chromatic scale should use sharps by default and convert to flats for flat keys.
worker/ssr/chords.js imports renderTabSSR from worker/utils.js. That function already exists — do not re-implement it.
main.js is written last. All showView('view-x') calls live in main.js. Router holds zero view names. Must handle /chords with registerRoute and /chords/:slug with registerPrefix. Final line must be boot().
index.html already has #view-chords and #view-chord-detail divs. Chords views target those existing IDs.
wrangler.toml env var for chords must be CHORDS_GID — matching the pattern in worker/sheets.js where the key is chords (all lowercase) and env vars follow UPPERCASE_GID convention.
Never redefine CSS custom properties in section CSS files. All :root tokens live exclusively in base.css.
