Gotchas for the Next Claude

js/views/chords.js is the biggest remaining file. It needs: list page render, detail page render, transpose engine, SVG diagram generator, popover logic (hover + tap), font-size control, auto-scroll, print button, share button. Build it in logical sections — don't try to write it as one monolith.
Transpose engine spec. Chromatic scale uses sharps by default: ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B']. Flat keys (F, Bb, Eb, Ab, Db, Gb) use flats for display. Parse chord name by splitting root from suffix: root is 1–2 chars (letter + optional #/b), suffix is everything after. Transpose root by ±semitones, reattach suffix unchanged. Range: −6 to +6.
SVG diagram renderer spec. Canvas: 80×100px. 6 strings vertical, 5 frets horizontal. String 0 (high-e) is rightmost, string 5 (low-E) is leftmost — matching chord-shapes.js array order. Nut = thick rect at top if baseFret === 1. Fret number label top-left if baseFret > 1. Open string = small circle above nut. Muted = × above nut. Finger dot = filled circle at intersection, finger number inside. Barre = filled rx rect spanning from→to strings at given fret. Colors: var(--accent) for dots/barre, var(--text) for strings/frets.
--tab-font-size CSS var. chords.js controls font size by doing tabBody.style.setProperty('--tab-font-size', newSize). Default .88rem. Step: .06rem. Min: .7rem. Max: 1.2rem. Save preference to localStorage key sd_tab_font_size.
Popover positioning on desktop. On mouseenter of a .chord-token, compute token.getBoundingClientRect(), then position popover left = rect.left + rect.width/2 - popoverWidth/2, top = rect.top - popoverHeight - 8. Clamp to viewport edges. On mobile (@media (hover: none)), CSS overrides position to centered — JS does not need to position it.
Auto-scroll. Uses setInterval calling window.scrollBy(0, 1) at an interval derived from the speed slider (1–10 → ~80ms to ~16ms). Stores interval ID, clears on toggle-off or route change. Speed slider value saved to localStorage key sd_scroll_speed.
worker/ssr/chords.js must import renderTabSSR from ../utils.js (already implemented there — do not rewrite). It renders the full chord detail page as static HTML for bots. MusicComposition schema. True HTTP 404 + noindex,nofollow for missing slugs.
main.js wires everything together. Use registerRoute('/', renderHome), registerRoute('/chords', renderChords) for the list, and registerPrefix('/chords/', renderChordDetail) for slugs (the prefix handler receives the slug as a parameter). registerPrefix('/blog/', renderArticle) for blog slugs. Final line must be boot().
wrangler.toml env var for chords is CHORDS_GID (uppercase, matching the worker/sheets.js convention where env.CHORDS_GID is read for the chords key).
index.html already has #view-chords and #view-chord-detail divs. chords.js renders the list into #view-chords and the detail into #view-chord-detail. Do not add new view divs.
updateSEO with chordMeta. For detail pages, pass chordMeta: { title, artist, key, excerpt, slug, imageUrl, tags } — seo.js uses this to inject a <script id="chord-schema" type="application/ld+json"> MusicComposition schema. removeSchemas() already knows to clean chord-schema.
